# Resume - 2026-02-21

Date: 21/02/2026 (February 21, 2026)

## Current Assessment

SEC EDGAR extraction produced wrong values for a relevant set of companies.

Main reasons observed:

1. Tag semantics mismatch across sectors.
Example: in insurance/broker models, what is economically "revenue" can appear under tags such as commissions and fees instead of generic revenue tags. This caused mapping inconsistencies across metrics.

2. Fiscal-year-end assignment issues.
BRO is a confirmed case where fiscal-year-end handling did not map periods correctly for expected calendar interpretation. Similar issues may exist for other tickers.

3. Broad reliability impact for pipeline base data.
Because these fundamentals feed the investment pipeline, current outputs cannot be treated as fully reliable at this stage.

4. Null coverage concerns.
Null analysis is tracked separately in `@sec_nulls....md`.

## Provider Status

1. SEC strategy: currently not reliable enough as implemented.
2. Alpha Vantage strategy: also problematic for quarter alignment in practice.
Example: requesting "Q1 2023" can return the fiscal quarter ending in a different calendar period (for example, CRWD fiscal cycles ending at/near January), creating calendar-vs-fiscal mismatches.
3. Financial data acquisition remains non-trivial due to fiscal/calendar reporting complexity and vendor-specific conventions.

## Conclusion

At this point, both the current SEC approach and the current Alpha Vantage approach are not viable as final production data foundations for the investment pipeline without further correction.

## Next Steps

1. Deepen fiscal-year vs calendar-year normalization design before provider migration.
2. Revisit metric taxonomy strategy (sector-aware tag selection, not only generic tags).
3. Evaluate low-cost paid providers, with Financial Modeling Prep as a candidate.
4. Define acceptance tests for quarter alignment and metric consistency before adopting any provider into production.
