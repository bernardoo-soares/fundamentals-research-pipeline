# Specs Tree Override

Applies inside `specs/`.

## Local Rules
1. Treat specs as executable contracts for code behavior and artifact schemas.
2. Keep specs aligned with current CLI commands, output paths, and column names.
3. When code changes schema/path/command behavior, update specs in the same change.
4. Mark status clearly with concrete dates for roadmap or phase notes.
5. Prefer additive, explicit edits over rewriting entire documents.
6. Do not describe unimplemented behavior as completed; call out gaps explicitly.
