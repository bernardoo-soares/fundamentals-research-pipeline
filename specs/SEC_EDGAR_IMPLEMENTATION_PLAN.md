# SEC EDGAR Pipeline Plan (Status as of 2026-02-21)

## 1. Objective
Maintain an auditable fundamentals pipeline for the current S&P 500 universe,
with SEC-first extensibility and deterministic quarterly keys.

Current code delivers universe, legacy canonicalization, SEC mapping, SEC raw
companyfacts ingestion, SEC submissions ingestion, fiscal calendar extraction,
SEC long normalization, and SEC processed yearly wide outputs. Wide canonical merge and ratio computation are
still pending.

### 2.3 Current SEC Normalization Scope
1. Filters by contract form/unit/tag priorities and year window.
2. Emits long canonical facts with source metadata.
3. Supports fiscal-quarter resolution from `period_end + fiscal_year_end_mmdd`
when fiscal calendar is provided.
4. Uses deterministic dedupe by key + latest `filed_date`/`accn`.
5. Includes unresolved fiscal-resolution artifact when strict mode is used.

## 3. Locked Policies
1. Universe policy: current S&P 500 constituents as-of run date.
2. Traceability policy: keep filing metadata and mapping version in outputs.
3. Key policy: deterministic quarterly keys centered on `ticker`, `fyearq`, `fqtr`.
4. Missing data policy: allow missing values; do not silently fabricate data.

## 4. Gaps to Close

into a wide quarterly canonical SEC table.
2. Merge SEC wide canonical rows with legacy canonical rows using explicit
precedence and conflict logs.
3. Recompute derived metrics/ratios from canonical raw fields only.
4. Add coverage and QA reports for publication readiness.
5. Introduce (or restore) a composite orchestration command after quality gates are in place.

## 5. Next Delivery Phases

### Phase A: Transform-Aware SEC Wide Canonicalization
Goal:
Convert long SEC facts into one row per `ticker,fyearq,fqtr` with audited
field-level lineage.

Deliverables:
1. SEC-wide canonical output (path to be finalized in implementation PR).
2. Conflict artifact for ambiguous same-key candidates.
3. Tests for Q4 extraction and fallback behavior.

### Phase B: Canonical Merge (Legacy + SEC)
Goal:
Merge legacy and SEC canonical tables with deterministic precedence and conflict
reporting.

Deliverables:
1. Merged canonical quarterly table.
2. Conflict log under `data/reports/`.
3. Tests for duplicate-key handling and precedence rules.

### Phase C: Derived Metrics and Reporting
Goal:
Compute derived screening metrics from raw canonical fields and publish QA
artifacts.

Deliverables:
1. Ratio/feature dataset(s).
2. Coverage and missingness reports.
3. End-to-end regression tests for reproducibility.

## 6. Validation Gate (Per Phase)
Required checks:
